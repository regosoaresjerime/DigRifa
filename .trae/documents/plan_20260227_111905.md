# Modernize Campaign Image Upload & Display

## 1. Analysis & Database Check
- **Database:** The `campaigns` table uses a text column `image` to store a JSON array of image URLs. This is compatible with the multi-image requirement (up to 3) without schema changes.
- **Current Bugs:**
  - **Delete Issue:** Likely caused by event bubbling or state synchronization issues. I will add explicit event prevention and robust state updates.
  - **Upload Issue:** The file input may be blocked by UI overlays or "stuck" loading states. I will restructure the component to ensure the input is always accessible.

## 2. Implementation Plan

### Phase 1: Campaign Creation Page (img 1) - `CampaignEditMedia.tsx`
- **Frontend Design Upgrade:** Apply the "Intentional Aesthetic" (Modern/Clean) using the `frontend-design` skill.
  - Use a card-based layout with clear hierarchy.
  - Improve the upload area with better drop-zone visuals.
  - Add smooth transitions for adding/removing images.
- **Functional Fixes:**
  - Limit uploads to **3 images** (currently 5).
  - Rewrite `handleImageUpload` to robustly handle the JSON array storage.
  - Fix the **Delete button**: Ensure `e.stopPropagation()` is used to prevent triggering other click events.
  - Fix the **Upload box**: Ensure the `<input type="file">` covers the entire clickable area and is not obstructed.

### Phase 2: Raffle Page (img 2) - `RafflePage.tsx`
- **Image Display Fix:**
  - Change the main image container to use `object-fit: contain` instead of `cover`.
  - This ensures images are **never cropped** and are shown in their entirety.
  - Add a blurred background or neutral fill to handle aspect ratio differences gracefully (e.g., a vertical image on a horizontal screen).
- **Carousel/Gallery:**
  - Ensure all 3 images are accessible via a gallery or slider if multiple are uploaded.
